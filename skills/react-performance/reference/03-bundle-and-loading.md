# 初期バンドルと読み込み戦略

## 優先度

CRITICAL

## 見る場所

- 画面の先頭import
- エディタ、チャート、地図、日付処理、解析SDK
- 初期表示では触らないダイアログや管理機能

## ルール

### 1. 初期表示に不要な重い依存は遅延ロードする

- 常時表示されない大型コンポーネントは `lazy()` や `next/dynamic` を使う
- 条件付きでしか使わない機能は、条件成立時に import する

### 2. 第三者ライブラリは重要度で分ける

- 解析、ログ、可視化、エディタは後ろへ送る候補
- フォーム入力や初期レイアウトに必須なら同期ロードのままでよい

### 3. バレルimportは無条件で肯定しない

- 共有UI配下の大きなバレルから引くと、不要なコードを巻き込みやすい
- 実装やビルド設定上の問題が見えているときは直接importへ寄せる

### 4. 先読みはユーザーの意図が見えるときだけ行う

- hover、focus、画面遷移直前など、次に使う可能性が高い場面だけプリロードする

## 指摘してよい条件

- 初期表示で使わない重いモジュールをトップレベルでimportしている
- まれにしか開かないダイアログやタブの依存が初回バンドルに載っている
- バンドルを重くする第三者SDKが初期描画前に動いている

## 指摘してはいけない条件

- 常時表示される小さなコンポーネント
- 分割するとUXや実装複雑度のほうが悪化する場合
- 直接importにしてもビルド結果が改善しない場合

## フレームワーク別メモ

- Next.js: `next/dynamic` を優先
- React SPA: `React.lazy` と `Suspense`、またはイベント内 `import()` を使う
