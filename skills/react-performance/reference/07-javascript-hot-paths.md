# JavaScript ホットパス

## 優先度

LOW-MEDIUM

## 前提

この領域は、描画や通信を直した後にまだ詰まる場合だけ触る。

## ルール

### 1. 繰り返し探索は `Set` `Map` を使う

- `includes` や `find` を大量反復するなら変換を検討する

### 2. 反復は必要な回数だけ回す

- `filter().map()` の連鎖や複数回走査は、ホットパスなら統合する

### 3. 早く終われる条件は先に判定する

- 配列長比較、null判定、ガード節で無駄な処理を減らす

### 4. ループ内で同じものを作らない

- 正規表現、派生キー、重い関数結果をループ外へ出す

### 5. min/max のために `sort()` しない

- 並び替えが不要なら1回の走査で求める

## 指摘してよい条件

- 大量データや高頻度更新の中で同じ探索を繰り返している
- ループ内に正規表現生成や重い計算がある

## 指摘してはいけない条件

- データ量が小さい
- 可読性を大きく落とすほどの微小改善しか見込めない
- 計測なしに全面的な書き換えを始める
