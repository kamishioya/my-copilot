# 描画とDOM負荷

## 優先度

MEDIUM

## ルール

### 1. 長い一覧は DOM 数を先に減らす

- 仮想化できるならそれが最優先
- 仮想化しない場合の補助として `content-visibility: auto` を検討する

### 2. 静的な要素や設定は毎回作り直さない

- 静的JSX、列定義、定数配列は外へ出せるか確認する

### 3. 条件描画は意図を明示する

- `count && ...` のような曖昧な条件より、表示条件を明示する

### 4. ハイドレーション問題はSSR時だけ扱う

- クライアント専用値でちらつく場合のみ対処する
- SPAにSSR向けの対策を持ち込まない

### 5. リソースヒントは本当に初期描画へ効くものだけ

- 主要フォント、ヒーロー画像、直後に遷移する画面などに限定する

## 指摘してよい条件

- 長大なリストをそのまま全描画している
- 毎renderで不変の配列やJSXを作り直している
- SSR画面でクライアント差分によるちらつきが起きる

## 指摘してはいけない条件

- 要素数が少なく、仮想化コストのほうが大きい
- SPAなのにSSR用のハイドレーション対策を入れる
- `content-visibility` をインタラクティブな小要素へ乱用する
